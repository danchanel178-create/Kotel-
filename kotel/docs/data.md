# Справочные данные и форматы файлов

## Каталог `data/reference/`

Используется `JsonFileDataProvider` при обычном запуске из `main.py`.

### `fuels.json` — справочник топлив

Структура:

```json
{
  "fuels": [
    {
      "id": "natural_gas",
      "name": "Природный газ (ГРС)",
      "fuel_type": "gaseous",
      "Q_i_p": 35588,
      "A_p": 0,
      "W_r_c": 0,
      "c_tl": 1.7,
      "CO2_carb": 0
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | string | Уникальный ключ пресета |
| `name` | string | Отображаемое имя |
| `fuel_type` | `"gaseous"` \| `"solid_liquid"` | Тип (для газов указывается явно; иначе обычно solid/liquid) |
| `Q_i_p` | number | Низшая теплота сгорания, кДж/кг или кДж/м³ |
| `A_p` | number | Зольность рабочей массы, % |
| `W_r_c` | number | Влажность, % |
| `c_tl` | number | Теплоёмкость топлива |
| `CO2_carb` | number | (CO₂)карб, % |

Пресеты в поставке: природный газ, ПНГ, СУГ, мазуты М-100/М-40, дизель, угли (Кузнецкий Г/Д, Экибастузский и др.), антрацит, бурый уголь, пеллеты, нефтяной кокс.

Чтобы добавить топливо — допишите объект в массив `fuels` и перезапустите приложение.

### `q5_nominal.json` — кривая рис. 5.1

Переопределяет встроенную `Q5_NOMINAL_CURVE` из `src/core/tables.py`.

Формат — пары (D_ном, q₅ном) для линейной интерполяции. Если файл отсутствует, используется встроенная кривая.

## Сессия расчёта (сохранение / загрузка)

Формат версии 2:

```json
{
  "version": 2,
  "fuel_preset_id": "kuzbass_g",
  "input": {
    "fuel": { "...": "..." },
    "ash": { "...": "..." },
    "flue": { "...": "..." },
    "boiler": { "...": "..." },
    "useful": { "...": "..." },
    "consumption": { "...": "..." }
  },
  "result": {
    "eta_k": 89.5,
    "B": 2.1
  }
}
```

- При **сохранении** всегда пишется `input`; `result` — только если расчёт уже выполнялся.
- При **загрузке** восстанавливаются поля формы и выбранный пресет; пересчёт нужно запустить вручную.

Файл можно править вручную, соблюдая имена полей из `HeatBalanceInput` / dataclass-моделей.

## Настройки UI — `settings.json`

В корне проекта:

```json
{
  "label_mode": "compact"
}
```

Управляется через окно настроек (`src/ui/settings_store.py`).

## Будущая БД
